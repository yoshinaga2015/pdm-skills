# 開発ノート

## 2026-02-13
- 今日は `skills/ui-creation` と `skills/ui-improvement` を新規追加した。
- 背景として、別ワークスペースで蓄積していた `Flow` のUI改善ノウハウを、再利用しやすい Agent Skill 形式に移植する必要があった。
- `ui-creation` では「新規画面を作るときの初期設計品質」を上げることを狙い、目的定義・情報階層・摩擦削減・信頼要素・A/B前提の出力フォーマットを固定化した。
- `ui-improvement` では「既存UIの改善速度」を上げることを狙い、課題分類から仮説生成、比較施策、計測設計、学習化までを一連の型として定義した。
- 特に意図したのは、CVR一点最適を避けること。両Skillともに継続品質（解約、返金、問い合わせ、体験劣化）をガードレールとして明示した。
- 後任向けメモ: 次に拡張するなら、`pricing` / `paywall` / `onboarding` の縦割り派生Skillを作ると、現場での呼び出し精度がさらに上がる。
